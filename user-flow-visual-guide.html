6Ô∏è‚É£ Notification Badge Behavior

‚è∞ 24-Hour Window Rule

Badges show items created in the last 24 hours only. This prevents overwhelming users with notifications for old content.

‚úÖ Badge Appears When:

  • ‚Ä¢ New action performed
  • ‚Ä¢ Item created < 24h ago
  • ‚Ä¢ User hasn't clicked tab yet
  • ‚Ä¢ Page loads/refreshes

‚ùå Badge Disappears When:

  • ‚Ä¢ User clicks the tab
  • ‚Ä¢ Item > 24h old (ages out)
  • ‚Ä¢ No new items exist
  • ‚Ä¢ User navigates away

üîÑ Badge Lifecycle Example

T+0h: Action performed ‚Üí Badge = 1 1
T+5min: User clicks tab ‚Üí Badge = 0 (cleared)
T+10min: User refreshes page ‚Üí Badge = 1 1 (item still < 24h)
T+24h: Item ages out ‚Üí Badge = 0 (permanently)
